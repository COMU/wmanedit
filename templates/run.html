NAME

The NAME section is the only required section. Man pages without a name section are as useful as refrigerators at the north pole. This section also has a standardized format consisting of a comma-separated list of program or function names, followed by a dash, followed by a short (usually one line) description of the functionality the program (or function, or file) is supposed to provide.

SYNOPSIS

The SYNOPSIS section is intended to give a short overview on available program options. For functions this sections lists corresponding include files and the prototype so the programmer knows the type and number of arguments as well as the return type.

DESCRIPTION

The DESCRIPTION section eloquently explains why your sequence of 0s and 1s is worth anything at all. Here's where you write down all your knowledge. This is the Hall Of Fame. Win other programmers' and users' admiration by making this section the source of reliable and detailed information. Explain what the arguments are for, the file format, what algorithms do the dirty jobs.

OPTIONS

The OPTIONS section gives a description of how each option affects program behaviour.

FILES

The FILES section lists files the program or function uses. For example, it lists configuration files, startup files, and files the program directly operates on.

ENVIRONMENT

The ENVIRONMENT section lists all environment variables that affect your program or function and tells how, of course. Most commonly the variables will hold pathnames, filenames or default options.

DIOGNASTICS

The DIAGNOSTICS section should give an overview of the most common error messages from your program and how to cope with them.

BUGS

The BUGS section should ideally be non-existent. If you're brave, you can describe here the limitations, known inconveniences and features that others may regard as misfeatures.

AUTHOR

The AUTHOR section is nice to have in case there are gross errors in the documentation or program behaviour and you want to mail a bug report.

SEE ALSO

The SEE ALSO section is a list of related man pages in alphabetical order. Conventionally, it is the last section. You are free to invent other sections if they really don't fit in one of those described so far.
